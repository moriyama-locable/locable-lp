Context

こんな時に、LOCABLEが入ります

デジタルコンテンツは、作るだけでは現場で成立しません。空間条件、機材、施工、導線、運営、保守までつながって初めて、リアルな体験になります。

01
映像・センサー・演出などのデジタル施策を、実空間にどう実装すればよいか分からない
02
コンテンツ制作会社、施工会社、設備会社、施設側の会話がかみ合っていない
03
企画は面白いが、現場条件・安全性・運用負荷を踏まえると成立するか不安がある
04
プロジェクター、センサー、PC、音響、配線、什器、内装の取り合いを整理する人がいない
05
開業・イベント本番までに、誰が何を決めるべきか曖昧なまま進んでいる
06
一度作って終わりではなく、運営しながら改善できる体制まで整えたい
What we do

主力は、実装とPM。
必要に応じて、企画構想までさかのぼります。

LOCABLEは、デジタルコンテンツをリアルな空間に実装するための翻訳者であり、推進役です。構想・制作・施工・運営の間に入り、現場で動く状態まで整えます。

Core Service
Implementation / PM

デジタルコンテンツ実装・開業PM

体験コンテンツを空間に実装するために、機材条件、施工条件、運営条件、関係者の役割分担を整理し、開業・本番までの推進を担います。

  • 機材・什器・配線・設置条件の整理
  • コンテンツ制作会社、施工会社、施設側の調整
  • スケジュール、課題、意思決定事項の管理
  • 現地確認、図面・レイアウト・運用条件の確認
Extension

企画構想・体験設計

実装の前提が曖昧な場合は、体験の狙い、対象者、導線、収益・運営との接続まで整理します。上流だけを語るのではなく、実装可能性を踏まえた構想に落とし込みます。

  • 体験コンセプトの整理
  • 成立条件と制約条件の可視化
  • 提案書・企画書の構造整理
  • 事業・運営への接続整理
実装条件の整理
プロジェクター、センサー、PC、音響、照明、配線、什器、施工の条件を現場目線で整理します。
関係者間の翻訳
制作会社、設計施工、施設運営、クライアントの間で、言葉と判断基準をそろえます。
プロジェクト推進
課題、スケジュール、決定事項を管理し、開業・本番まで止まらない状態をつくります。
現地・図面確認
現地条件や図面を見ながら、設置・導線・安全・保守に関わる論点を洗い出します。
運営条件への接続
開業後にスタッフが運用できるか、トラブル時に対応できるかまで含めて整えます。
企画構想の補強
必要な場合は、体験の狙い・収益・再来訪・売場接続までさかのぼって整理します。

How to start

3つの頼み方

入口は、実装PMから。必要に応じて、企画構想や事業設計まで広げていく形が基本です。

01
Core
実装PM支援
Project PM

デジタルコンテンツを実空間に導入する案件に入り、制作・施工・施設運営の間をつなぎます。開業・イベント本番・設置完了まで、課題整理と進行管理を担います。

向いている状況導入する内容は決まっているが、現場実装と関係者調整に不安がある
02
実装前レビュー
Spot Review

提案書、図面、機材構成、現地条件を確認し、実装上のリスクや不足論点を洗い出します。提案前・発注前・現地調査後の確認に向いています。

向いている状況大きな契約や工事に入る前に、破綻しやすい論点を確認したい
03
企画構想支援
Upstream

まだ何を作るかが固まっていない段階から、体験の狙い、来場者行動、収益・運営との接続、実装可能性を整理します。実装を前提にした上流整理です。

向いている状況場所やテーマはあるが、体験・事業・実装条件がまだつながっていない
Scope

担うこと / 担わないこと

実装PMの価値を曖昧にしないため、責任範囲を分けて考えます。必要に応じて、専門会社とチームを組みます。

LOCABLEが担うこと
体験コンテンツの実装条件整理
関係者間の要件整理・論点整理
進行管理、課題管理、意思決定支援
現地条件・図面・運営条件の確認
実装を前提にした企画構想の整理
原則として専門会社と分担すること
コンテンツそのものの詳細開発・プログラム制作
電気・内装・設備工事の施工責任
法定設計・確認申請などの設計者責任
機材メーカーとしての保証・保守
広告運用や集客施策の実務代行

※案件ごとに責任範囲を整理し、必要な専門会社・制作会社・施工会社と連携して進めます。
